# Default-allow for nearly everything. The only path we exclude is the
# JSON data API at /api/v1/* — search engines should index the HTML
# pages (/block/N, /contract/0x...) not raw JSON. The /api/og/*
# OG-card route is implicitly allowed so social-preview scrapers
# (Twitter, Discord, Telegram, Slack, LinkedIn, Facebook) can always
# fetch link-preview images.
#
# We learned at launch that some robots.txt parsers (notably Twitter's)
# don't implement the spec's "most-specific-match-wins" rule correctly:
# they read top-to-bottom and treat a broad `Disallow: /api/` as
# blocking everything under it, including /api/og/. Writing the rules
# without overlapping patterns sidesteps that bug.
#
# Per-bot explicit allows below are defense-in-depth: per the robots.txt
# spec, a crawler matching a specific User-agent block uses ONLY that
# block's rules and ignores the wildcard. So a bot listed here is fully
# governed by its own "Allow: /" and can't be confused by anything in
# the wildcard section.
